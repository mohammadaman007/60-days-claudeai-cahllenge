🧠 Architecture Overview

This studio runs a genuine multi-agent loop against the live Claude API. Each agent is a distinct system prompt with a narrow job. The Evaluator → Critic → Improver triad repeats inside a real while loop — the number of rounds is not fixed in advance. Every round, the loop checks three stop conditions in order: (1) plateau — score improved less than the configured delta for two consecutive rounds, (2) threshold — score crosses the target set at intake, (3) hard cap — a safety fallback so the loop can never run away. Whichever fires first ends the loop, and the exact reason is surfaced in the final summary.

State threads forward explicitly: each Improver call receives the prior draft, the Evaluator's numeric rubric + reasoning, and the Critic's critique. Each Evaluator call receives the current draft and the fixed rubric. A Memory Manager distills durable lessons each round (e.g. "avoid passive voice in intros") which get folded into the next Improver prompt, and a Safety Monitor screens the near-final draft before the Final Reviewer signs off.

Agent Roles

Planner — builds outline, angle, keyword targets Executor — writes the first full draft Evaluator — scores quality/SEO/engagement with reasoning Critic — finds concrete weaknesses the score alone hides Improver — rewrites using evaluation + critique Memory Manager — distills durable lessons across rounds Safety Monitor — screens for policy/safety/factual issues Final Reviewer — polish pass + approval summary

Extension Ideas

• Swap in a real SEO API (e.g. keyword volume) to ground the Evaluator's SEO sub-score in data rather than model judgment.
• Add a Human-in-the-loop gate before Final Reviewer for regulated content.
• Persist Memory Manager output across sessions (not just rounds) so brand voice learnings compound over many briefs.
• Add a Fact-Checker agent with web search tool access for claims-heavy content.
• Parallelize: run 2 Improver variants per round and let the Evaluator pick the winner (tournament-style).
