📚 Key Learnings from the "Defend Your Experience" Prompt

🎯 Interview-first approach: Understand the user's goal before building any solution.

❓ One question at a time: Gather information progressively using MCQs for a natural interview experience.

👥 Audience awareness: Tailor the experience based on whether the user is preparing for HR, technical, managerial, or panel interviews.

📄 Evidence-driven analysis: Extract every meaningful claim from the uploaded resume and treat each as something that must be defended.

🤖 Adaptive AI conversations: Generate personalized follow-up questions where each answer influences the next challenge.

🔍 Critical thinking over resume review: Focus on validating claims, identifying weak evidence, and improving confidence rather than rewriting the resume.

📊 Insightful analytics: Use confidence scores, progress tracking, visualizations, and defense metrics to measure preparedness.

💡 Actionable feedback: Produce a Defense Report highlighting strengths, weak areas, supporting evidence, and recommendations.

🎨 User-centric UX: Adapt the interface, typography, layout, and interactions to the user's preferred visual style instead of using a generic design.

💾 Production-ready features: Include drag-and-drop uploads, local storage, session history, exports, responsive design, and graceful API error handling.

🚀 Modern AI integration: Build a fully client-side application using the Anthropic Messages API without requiring API keys or a backend.

🏆 Real interview readiness: Help users confidently defend every experience and achievement with clear, evidence-backed responses during actual interviews.
