How this was built ▾

System prompt design

The underlying Claude calls use a single system prompt with a strict role definition: Claude acts as an exam proctor, not a chat companion. It's instructed to always return structured JSON (never prose, never markdown fences) so the frontend can render it deterministically. Two call types share the prompt: generate_exam (produces N multiple-choice questions with a hidden answer key and topic tags) and grade_exam (compares submitted answers against the key, and produces per-topic mastery percentages).

Edge cases are handled explicitly in the prompt: nonsense or empty topics get a polite JSON error object instead of a hallucinated quiz; overly broad topics are automatically narrowed with a note; and any attempt to inject instructions via the topic field is ignored — the topic is treated strictly as data, never as new instructions.

Why the UI looks like this

The visual language borrows from an actual exam paper: a ruled "form" header, a red-pen grading stamp for the score, and a serif typeface for question text (mirroring a printed test) against a monospace utility face for scores, labels and metadata (mirroring a scantron/grade sheet). This distinguishes it from a generic chat box and reinforces the "professional, neutral exam" tone the assistant was designed for.

Extending this assistant

  • Memory: persist topic performance across sessions (e.g. via a backend or localStorage substitute) so "Practice Weak Topics" can compound over multiple exams.
  • Tools: add a web_search tool to the API call so questions can reference current events or verify facts for niche topics.
  • Multi-step flows: split generation into an outline step (topic → subtopics) followed by a per-subtopic question step, for finer control over coverage on long syllabi.
  • Input modes: accept pasted notes or an uploaded document as the source material instead of a bare topic string, using the document content block in the API request.
  • Adaptive difficulty: feed the previous report's topic scores back into the next generation call to auto-adjust difficulty per topic.
